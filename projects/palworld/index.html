E · SYSTEM DECOUPLING

多系统耦合处理思考

三层解耦架构 + 五大策略，从设计到前端建立边界规则
系统耦合防范措施
从架构设计到前端落地，建立多层解耦防御体系
⚠ 为什么大地图容易产生耦合问题
大地图是游戏中系统耦合最多的界面——任务、副本、收集、BOSS、活动、公会…… 所有系统都想在地图上「插一脚」。如果没有边界规则，最终会变成一锅粥： 改一个系统牵一发而动全身，新功能接入要改多处代码，出问题不知道谁的锅。
01
数据耦合
各系统直接读写地图数据，没有统一数据入口
02
逻辑耦合
地图UI里直接写各系统的业务逻辑，越堆越乱
03
表现耦合
各系统点位样式不统一，各写各的渲染逻辑
🏗 三层解耦架构
L3 表现层 UMG Widget / Lua UI Component
组件化UI · 统一点位基类 · 样式配置化 · 状态机驱动显隐
↓ 通过统一接口调用 ↓
L2 逻辑层 MapSystem / EventBus
事件总线通信 · 统一数据协议 · 优先级调度 · 生命周期管理
↓ 通过数据层读写 ↓
L1 数据层 MapData / Config
配置表驱动 · 数据订阅机制 · 只读访问 · 状态与表现分离
🛡 五大解耦策略
01 配置化驱动
点位信息、筛选规则、显隐逻辑全部走配置表，新系统接入不需要改地图核心代码
落地方式
配表声明 → 自动注册 → 即时生效
02 事件总线通信
各系统不直接调用地图接口，通过事件总线发布订阅，系统间完全解耦
落地方式
发布事件 → 总线转发 → 地图响应
03 组件化UI
统一点位基类，公共逻辑（缩放、选中、tooltip）基类实现，业务系统只扩展差异
落地方式
继承基类 → 覆写差异 → 自动复用
04 数据订阅机制
数据层统一管理，UI层订阅数据变化。谁改数据谁通知，UI只负责响应刷新
落地方式
数据变更 → 自动通知 → UI刷新
05 优先级调度器
点位显隐、动效、加载统一由调度器管理，避免各系统各自控制导致层级冲突
落地方式
注册申请 → 调度排序 → 统一执行
💡 实践案例：区域探索功能接入
接入前的耦合风险
区域探索涉及声望、商栈、Boss、公会4个子系统，如果各子系统直接往地图塞UI， 会导致地图代码里混入大量探索业务逻辑，后续改动任何子系统都要改地图
解耦后的接入方式
① 各子系统独立维护自己的点位数据，通过配表声明类型和优先级 ② 统一通过EventBus通知地图增删点位，不直接操作地图Widget ③ 探索详情窗作为独立模块接入右侧窗，通过ModuleManager动态加载卸载 ④ 各子系统只负责提供数据，地图只负责渲染和交互，职责清晰
效果
新增子系统不需要修改地图核心代码，配表 + 注册事件即可接入； 地图出bug时可以快速定位是数据层、逻辑层还是表现层的问题，排查效率提升

玩法入口与引导设计

玩家视角 · PLAYER'S JOURNEY
你在帕鲁世界里悠闲地探索，突然发现远处有一座神秘的古老高塔。 走近时，NPC凑过来告诉你传说中的宝藏。带着好奇，你踏入了塔内—— 就这样，你"自然而然"地开始了第一次副本挑战。
✨

🎭

🏰

🎉
活动限时入口

活动推送 → 活动中心 → 快捷跳转 → 参与玩法
🗺️
大地图点位入口
大地图上的副本点位直接点击进入，配合标记与追踪功能，降低寻找成本
地图浏览 → 点位点击 → 详情预览 → 前往/匹配
叙事包裹教学
好的引导不是"教玩家怎么玩"，而是"让玩家在完成目标的过程中自然学会玩法"。将教程嵌入剧情任务，而非独立出来。
多入口 ≠ 乱入口
每个入口都有明确的场景和用户心智：主线是"第一次遇见"，地图是"主动探索"，活动是"限时机会"。

F-4 · SETTLEMENT & EXIT

结算与退出机制

胜利要让你想炫耀，失败要让你想再来一局

情绪曲线 · EMOTION CURVE
Boss倒下的瞬间——胜利！ 音乐切换到凯旋的旋律，宝箱从空中缓缓落下，金光闪闪。 你屏住呼吸看着宝箱开启……"哇，出了稀有装备！" 这种从紧张到释放的情绪落差，就是玩家愿意一次又一次挑战的原因。

胜利结算
VICTORY
  • 胜利演出动画 + 音效
  • 宝箱开启（普通/稀有差异化表现）
  • 评分评级 + MVP评选
  • 奖励物品列表 + 稀有度展示
  • 战斗统计（伤害/承伤/治疗）
  • 再挑战 / 返回选项
失败结算
DEFEAT
  • 失败表现（不挫败、鼓励再试）
  • 失败原因提示（建议改进方向）
  • 已获得的保底奖励
  • 快速重试选项
  • 难度调整建议
  • 返回 / 组队调整选项
成就感的节奏感
结算不是"一张面板"，而是一段体验：从胜利的瞬间冲击，到宝箱的期待感，再到看到奖励的满足感——每一步都有情绪递进。
退出不焦虑
倒计时退出给玩家"从容看完奖励"的时间，同时自动退出避免挂机。进度保存机制让"下次再来"没有心理负担。
F-5 · DUNGEON SYSTEM

多副本类型体系

每种玩家都能找到属于自己的挑战方式

玩家画像 · PLAYER PERSONA
有人喜欢每天刷日常获取材料，有人热衷于挑战高塔追求极限； 有人喜欢探索洞窟发现秘密，有人享受世界Boss的热闹…… 好的副本体系，不是让所有玩家玩同一种玩法， 而是让每种玩家都能在这个世界里找到自己的乐趣。
🕳️
洞窟副本
日常挑战 · 材料获取
即日常副本（1.0命名），短平快的日常挑战提供稳定养成材料产出，洞窟内多路径探索+机关解谜强化探索感
每日重置 多难度选择 多路径探索 组队/单人
🗿
密域Boss
区域探索 · 精英挑战
分布在大世界各区域的Boss点位，探索过程中发现并挑战，是区域探索的目标点
大地图点位 多属性Boss 挑战机制 多人组队
🌍
世界BOSS
多人同屏 · 全民参与
大型世界Boss玩法，多人同屏挑战，参与即有奖，排名越高奖励越丰厚
多人同屏 排名奖励 限时开放 参与奖
🗼
高塔挑战
爬塔Roguelike · 高难
多层级爬塔玩法，越往上难度越高、奖励越好，是硬核玩家的长期挑战目标
层级递进 排行榜 MVP评选 持续挑战
🎭
活动副本
限时活动 · 主题玩法
与活动绑定的限时副本，每期有不同主题和机制，提供新鲜感和限定奖励
限时开放 主题机制 限定奖励 活动剧情
统一入口框架 统一结算流程 差异化体验 覆盖不同需求 长期挑战目标 碎片化适配 社交展示 探索乐趣
统一框架，差异化体验
所有副本共用一套入口、结算、匹配的基础框架，但每个类型有自己的特色机制和节奏，兼顾效率与新鲜感。
覆盖不同需求
洞窟（日常）满足材料需求，密域绑定探索，世界Boss做社交，高塔做硬核，活动做新鲜感——每种玩家都能找到适合的玩法。
